Schematics

Here to help people to have a nice project scaffolding and Nest architecture component generator, the @nestjs/schematics library can help more than expected. The purpose is to scaffold generic projects with a standard Nest structure.
The @nestjs/schematics library is based on the @angular-devkit projects.

Usages

You can use @nestjs/schematics with the angular-devkit tools or integrated with the @nestjs/cli tool.

ATTENTION! This section will expose how to use the @nestjs/schematics only.

Installation

You need first to install the @angular-devkit/schematics-cli globally first : 

    
      $ npm install -g @angular-devkit/schematics-cli

Next you have to git clone and build the @nestjs/schematics :

Docker 
    
      $ git clone https://github.com/nestjs/schematics.git
      $ cd schematics
      $ make prepare
      $ make build

Run a Schematic

Go to your workspace directory and execute the following command : 

    
      $ schematics [path_to_the_built_nestjs_schematics_folder]:[nestjs_schematic_name] [--options...]

To use the right schematic with the right options refer to the @nestjs/schematic collection section
