Usages

This section explain how to use NestjsCLI.

Commands

    
      $ nest [command] [...options]

Options

Each command can accept the list of options below:

  • --dry-run : allow to simulate the command execution to see how it will affect your working directory.

new

The new command generates a standard Nestjs project based on the typescript-starter as well as install the packages from npm or yarn package manager. NestjsCLI will ask you for missing information to create the project as which package manager you want to use for install part.

Option Name Description Required Default Value
name Your application name false nestjs-app-name
description Your application description false description
version Your application version false 1.0.0
author Your application author false ''

generate

The generate command generates a Nestjs architecture component.

Option Name Description Required Default Value
schematic The schematic name from the list bellow. true N/A
name The name of the generated Nestjs architecture component. true N/A
path The path from to generate the Nestjs architecture component. false src

Manage architecture component

  • controller
  • exception
  • guard
  • interceptor
  • middleware
  • module
  • pipe
  • service
